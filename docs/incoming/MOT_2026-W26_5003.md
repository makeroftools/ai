### Ask 4: Supabase.WeOwn.Dev — Still Relevant?

**Yes — EmDash strengthens the argument, not weakens it.**

EmDash is a CMS. Supabase/pgvector is a VectorDB. They solve different things:

```
EmDash              ← "Where content lives and is served"
  └─ MCP → AnythingLLM
              └─ LanceDB (embedded, per-instance, fast)
              └─ Supabase/pgvector (external, shared, scalable)
```

| Factor | LanceDB (current) | Supabase/pgvector (target) |
|--------|-------------------|---------------------------|
| **Scope** | Per-instance | Cross-instance, shared knowledge |
| **Scale** | Embedded (single container) | External (scalable) |
| **Query** | AnythingLLM native | SQL + vector search |
| **Use case** | Per-workspace RAG | Cross-product knowledge graph |

**Recommendation:** EmDash first (quick win, visible product), Supabase second
(infrastructure, longer lead). They don't conflict — EmDash is customer-facing,
Supabase is backend depth.
